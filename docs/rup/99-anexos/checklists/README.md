# Checklists de Qualidade e Processos

Este diretório reúne os checklists temáticos utilizados para orientar revisões de qualidade, implementação e operação do projeto Brikaya. Cada arquivo é numerado para facilitar o rastreamento e cobre uma dimensão específica do ciclo de vida do jogo.

## Arquivos disponíveis

- `001-governanca-operacional-checklist.md`: orientações sobre políticas operacionais e requisitos de qualidade.
- `002-visao-estrategica-checklist.md`: checklist voltado à visão do produto, alinhamento de objetivos e métricas de sucesso.
- `003-arquitetura-plataforma-checklist.md`: itens para avaliar arquitetura do jogo, performance e requisitos técnicos.
- `004-design-experiencia-checklist.md`: verificações relacionadas à experiência do jogador, interface e acessibilidade.
- `005-planejamento-governanca-checklist.md`: passos para planejamento e coordenação entre desenvolvimento e qualidade.
- `006-implementacao-padroes-checklist.md`: padrões de código TypeScript/React, qualidade e aderência a convenções.
- `007-testes-validacao-checklist.md`: diretrizes para testes de gameplay, performance e validação de funcionalidades.
- `008-entrega-implantacao-checklist.md`: requisitos para builds PWA, releases nativos e distribuição.
- `009-governanca-tecnica-checklist.md`: controles de governança técnica, gestão de dependências e conformidade.
- `010-ux-brand-checklist.md`: itens de verificação para consistência visual e diretrizes de UX para jogos.
- `011-contribuicao-processos-checklist.md`: orientações para contribuições e integração de novos recursos.
- `012-agentes-ia-checklist.md`: checklist para uso de agentes de IA no desenvolvimento do jogo.
- `013-documentacao-legada-checklist.md`: diretrizes para manutenção e atualização de documentação.

Cada checklist pode ser aplicado isoladamente ou em conjunto, dependendo do escopo da atividade em revisão.
