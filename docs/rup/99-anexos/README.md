# Anexos e Referências

Este diretório contém materiais de apoio, checklists de qualidade e referências técnicas utilizadas no desenvolvimento do Brikaya.

## Estrutura

### Checklists
Diretório `checklists/` contém checklists temáticos para garantir a qualidade em diferentes aspectos do projeto:

- Governança e processos
- Qualidade de código  
- Testes e validação
- UX e acessibilidade
- Documentação

### Referências Técnicas
Diretório `reference/` contém bibliotecas e documentação de apoio:

- **Heroicons:** Biblioteca de ícones para interface
- **Swagger:** Referência para documentação de APIs

## Como utilizar

Os checklists devem ser utilizados durante as revisões de código, planejamento de sprints e validação de releases. Cada checklist é numerado para facilitar o rastreamento e pode ser aplicado isoladamente ou em conjunto.

As referências técnicas servem como base para implementações e devem ser consultadas durante o desenvolvimento de novos componentes ou funcionalidades.
